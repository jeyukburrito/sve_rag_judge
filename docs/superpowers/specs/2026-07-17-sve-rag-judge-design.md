# SVE RAG 룰 저지 — 설계 문서

날짜: 2026-07-17
상태: 승인됨 (1단계 범위)

## 목적

Shadowverse Evolve 종합 룰북(일본어, v1.26.1)에 대해 한국어로 룰 질문을 하면, 공식 룰 조항을 근거로 인용하며 답변하는 로컬 CLI 도구.

- 1단계 (이 설계): 룰 Q&A CLI
- 2단계 (이후): 카드 데이터 추가 → 게임 상황 판정 저지, 웹/메신저 인터페이스

## 제약 및 원칙

- 전부 무료/오픈소스/로컬 실행 (API 비용 없음)
- 모든 답변은 룰 번호 인용 필수. 근거를 찾지 못하면 "모른다"고 답한다 — 확신에 찬 오답보다 낫다
- 벡터 저장소는 ChromaDB. LangChain 사용은 허용되나 필수 아님 (1단계 구현은 chromadb/ollama 클라이언트 직접 호출로 충분하면 직접 호출 우선)

## 아키텍처

```
[오프라인 인덱싱 — 1회]
룰북 PDF → (기존 scripts/ 파이프라인) 텍스트 추출 → 조항 단위 청킹 → bge-m3 임베딩 → ChromaDB 저장

[온라인 질의 — CLI]
한국어 질문 → bge-m3 임베딩 → ChromaDB top-k 검색 → 프롬프트 조립 → Ollama LLM → 근거 인용 답변
```

- **임베딩**: `bge-m3` (다국어 — 한국어 질문 ↔ 일본어 원문 교차 매칭), Ollama 임베딩 API로 실행. LLM과 런타임 통일, torch 설치 불필요
- **LLM**: Ollama 로컬 모델. 기본값 qwen3 계열(한/일 처리 양호), 설정으로 교체 가능
- **ChromaDB**: 로컬 persistent 모드. 서버 불필요

## 파일 구성

```
sve_rag_judge/
├── scripts/                  # 기존 PDF 추출 스크립트 (변경 없음)
├── data/                     # 룰북 PDF (변경 없음)
├── src/
│   ├── chunk_rules.py        # 추출 텍스트 → 조항 단위 청크 JSONL
│   ├── build_index.py        # 청크 JSONL → 임베딩 → ChromaDB
│   └── judge_cli.py          # 질문 → 검색 → Ollama 답변 (메인 CLI)
├── tests/
│   └── test_chunking.py      # 청킹 로직 단위 테스트
├── index/                    # ChromaDB 저장소 (.gitignore)
└── requirements.txt          # chromadb, ollama
```

## 컴포넌트 상세

### chunk_rules.py
- 조항 번호 정규식(`^\d+(\.\d+)*\.?`)으로 분리
- 청크 형식: `{"rule_id": "5.3.2", "text": "일본어 원문", "parent": "5.3"}`
- 제목만 있는 등 너무 짧은 조항은 부모 조항에 병합

### build_index.py
- 청크마다 Ollama `bge-m3` 임베딩 → ChromaDB 컬렉션 `rules` (문서=원문, 메타데이터=rule_id)
- 재실행 시 컬렉션 삭제 후 전체 재구축 (룰북은 통째로 개정되므로 증분 업데이트 없음)

### judge_cli.py
- 대화 루프: 질문 임베딩 → top-k(기본 8) 검색 → 프롬프트 조립 → 답변 출력
- 프롬프트 규칙: 검색된 조항만 근거로 한국어 답변, `[룰 X.X.X]` 형식 인용 필수, 근거 부족 시 "룰북에서 근거를 찾지 못했습니다"
- 답변 뒤에 참조 조항 원문 표시 (사용자 직접 검증용)

## 에러 처리 (최소)

- Ollama 미기동/모델 미설치 → 시작 시 체크, 설치 명령 안내 후 종료
- 검색 유사도 전부 낮음 → LLM 호출 없이 "관련 조항을 찾지 못했습니다" 반환 (환각 1차 방어)

## 테스트

- 자동: `tests/test_chunking.py` — 샘플 텍스트로 조항 분리와 rule_id 정확성 assert (LLM 의존 없음)
- 수동 성공 기준:
  - 인덱싱 후 청크 수/샘플 육안 확인
  - 룰 질문 5~10개: 인용 존재 여부, 인용 조항이 답변을 실제 뒷받침하는지, 룰북 외 질문에 "근거 없음" 응답하는지
