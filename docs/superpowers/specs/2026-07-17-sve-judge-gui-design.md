# SVE 룰 저지 GUI — 설계 문서

날짜: 2026-07-17
상태: 승인됨

## 목적

1단계 룰 Q&A CLI를 브라우저에서 쓸 수 있는 간단한 채팅형 GUI로 감싼다. 2단계(판정 저지) 진입 전의 경량 래퍼이며, 새 RAG 로직은 만들지 않는다.

## 설계

- **파일**: `src/judge_gui.py` 하나
- **프레임워크**: Gradio (`gr.ChatInterface`). `requirements.txt`에 `gradio` 한 줄 추가
- **로직 재사용**: `src.judge_cli`의 `ensure_ollama`, `retrieve`, `answer`, `DEFAULT_DB`, `EMBED_MODEL`, `DEFAULT_LLM`, `MAX_DISTANCE`를 import — 검색/프롬프트/게이트 로직 중복 금지
- **응답 형식**: 답변 텍스트 + 구분선 + "참조 조항" 목록(룰 번호, 거리, 원문 앞부분). 대화 히스토리는 화면 표시용일 뿐 LLM에 전달하지 않는다 (CLI와 동일한 단발 질의)
- **게이트**: CLI와 동일 — 거리 게이트(MAX_DISTANCE) 통과 조항이 없으면 LLM 호출 없이 "관련 조항을 찾지 못했습니다"
- **실행**: `python -m src.judge_gui [--db] [--llm] [--port 7860]` → 127.0.0.1 전용, 브라우저 자동 오픈(`inbrowser=True`)
- **시작 체크**: Ollama 연결/모델 존재/빈 인덱스 — 기존 가드 재사용, 실패 시 명확한 메시지로 종료
- **질의 중 예외**: 채팅창에 에러 메시지 문자열로 표시 (프로세스는 유지)

## 테스트

- 자동 테스트 없음 (UI 래퍼일 뿐, 로직은 기존 테스트가 커버)
- 수동 검증: 룰 질문 1개(인용 + 참조 조항 표시 확인), 무관 질문 1개(거부 확인)

## 의도적으로 뺀 것

대화 컨텍스트 유지, 외부 공개(share), 스트리밍 출력, 카드/판정 기능 — 2단계에서 판단.
